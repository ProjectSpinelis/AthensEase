Class Documentation: UI_Screen3

This document provides an overview of the UI_Screen3 JavaFX class, which represents the third screen in the trip questionnaire application.

Class Overview

UI_Screen3 is a JavaFX application class that prompts the user to input their trailhead information and determines the next step in the application based on whether the user is traveling on a budget.

Fields

  • String trailhead: Stores the user-entered trailhead information.
  • boolean isBudgetYesSelected: Indicates whether the user is traveling on a budget. This value is passed from the previous screen.

Methods

setIsYesBudget(boolean isBudgetYesSelected)

Setter method to initialize the isBudgetYesSelected field with the user's budget preference passed from the previous screen.

public void setIsYesBudget(boolean isBudgetYesSelected)

start(Stage primaryStage)

The main entry point for the JavaFX application. Sets up the user interface and handles user input, validation, and navigation logic.

Key Responsibilities:

  • Prompts the user to input their trailhead using a text field.
  • Handles input validation when the Submit button is clicked.
  • Provides a Continue button that determines the next screen based on the user's budget preference.
public void start(Stage primaryStage)

User Interaction

  • The user enters their trailhead in the text field and clicks Submit. Input is displayed in the console for verification.
  • The Continue button navigates the user to the next screen based on the isBudgetYesSelected value:
    • If true, transitions to UI_Screen5.
    • If false, displays a completion message using an alert dialog and closes the application.

Validation

Input validation is performed when the Submit button is clicked:

  • The trailhead input must not be empty or null.
  • If validation fails, a message is displayed in the console, and the program does not proceed.

Navigation Logic

  • If isBudgetYesSelected == true, creates an instance of UI_Screen5 and passes the budget preference.
  • If isBudgetYesSelected == false, displays a completion alert dialog and closes the application window.

UI Layout

The user interface consists of the following components:

  • Top Section: Contains a title, text field for trailhead input, and a Submit button.
  • Bottom Section: Contains a Continue button positioned at the bottom-right of the screen.

The layout uses a BorderPane for clear separation of sections.
