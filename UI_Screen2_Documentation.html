Class Documentation: UI_Screen2

This document provides an overview of the UI_Screen2 JavaFX class, which represents the second screen in the trip questionnaire application.

Class Overview

UI_Screen2 is a JavaFX application class that collects user input for trip-related information, including:

  • The number of days the user will spend in Athens.
  • Whether the trip involves one or multiple trailheads.
  • Whether the user is traveling on a budget.

The class transitions to subsequent screens based on user input.

Fields

  • int duration: Stores the number of days the user will spend in Athens. Default value is -1 to indicate invalid input.
  • Boolean isOneTrailheadSelected: Tracks the user's selection for the trailhead question. Initially null.
  • Boolean isBudgetYesSelected: Tracks the user's selection for the budget question. Initially null.

Methods

start(Stage primaryStage)

The main entry point for the JavaFX application. Sets up the user interface and handles input validation and navigation logic.

Key Responsibilities:

  • Displays three questions to the user, each with specific input types (text field or buttons).
  • Validates input when the Continue button is clicked.
  • Transitions to either UI_Screen3 or UI_Screen4_1 based on the user's responses.

User Interaction

  • Question 1: Text field input for the number of days in Athens. Input is validated to ensure it is an integer.
  • Question 2: Two buttons (One or Many) for selecting the number of trailheads.
  • Question 3: Two buttons (Yes or No) for selecting whether the user is traveling on a budget.
  • The Continue button validates all inputs before transitioning to the next screen.

Navigation Logic

  • If isOneTrailheadSelected == true, transitions to UI_Screen3.
  • If isOneTrailheadSelected == false, transitions to UI_Screen4_1, passing the duration and isBudgetYesSelected values.
  • Displays validation messages if any required input is missing.

Validation

The following validations are performed when the Continue button is clicked:

  • duration: Must be a valid integer.
  • isOneTrailheadSelected: Cannot be null.
  • isBudgetYesSelected: Cannot be null.

If any validation fails, the program displays an appropriate message in the console and does not proceed to the next screen.
