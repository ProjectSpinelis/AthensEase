Class Documentation: UI_Screen4_1

The UI_Screen4_1 class is a JavaFX screen that allows the user to specify the days when they will change their trailhead during their trip.

Class Overview

This screen prompts the user to input days where the starting point changes during a multi-day trip. It ensures that the first day is always included and validates user input for correctness.

Fields

  • int duration: Total number of days in the trip, passed from a previous screen.
  • List<Integer> changeDays: A list to store days of trailhead changes, starting with day 1.
  • boolean isBudgetYesSelected: Indicates whether the user is traveling on a budget.

Constructor

The constructor initializes the duration and isBudgetYesSelected fields, passing data from the previous screen.

public UI_Screen4_1(int duration, boolean isBudgetYesSelected)

Methods

start(Stage primaryStage)

The main entry point for the screen. Sets up the user interface and handles the following functionality:

  • Displays the duration of the trip.
  • Prompts the user to input days for trailhead changes using a comma-separated text field.
  • Validates input to ensure all days are within valid range and not duplicated.
  • Transitions to UI_Screen4_2 with the list of change days.
public void start(Stage primaryStage)

User Interaction

  • The user enters days in a comma-separated format in the text field (e.g., "2, 4").
  • The input is validated, ensuring:
    • All days are integers.
    • Days are within the range of 2 to duration.
    • Day 1 is always included.
  • Clicking Submit displays the list of days in the console.
  • Clicking Continue transitions to UI_Screen4_2, passing the validated list of change days and the budget preference.

Validation

Validation logic ensures that:

  • Input is parsed into integers.
  • Only days within the valid range (2 to duration) are accepted.
  • Duplicates are excluded.
  • If validation fails, the list resets to contain only day 1.

Navigation Logic

  • On Submit: Prints the list of days and the total number of trailhead changes.
  • On Continue: Transitions to UI_Screen4_2, passing:
    • The list of change days (always including day 1).
    • The budget preference (isBudgetYesSelected).

UI Layout

The user interface is designed using the following JavaFX components:

  • VBox: Contains labels for instructions and the text field for input.
  • BorderPane: Organizes the layout with:
    • Center: The main content (instructions, input field, and submit button).
    • Bottom: A Continue button aligned to the bottom-right.
