Class Documentation: UI_Screen5

The UI_Screen5 class is a JavaFX screen where users input their budget and either proceed to the next step or complete the process.

Class Overview

This screen allows the user to provide a budget in euros. It validates the input, displays feedback, and offers a continuation button to proceed or complete the process.

Fields

  • boolean isBudgetYesSelected: Indicates if the user selected "Yes" to traveling on a budget in previous steps.
  • int budget: Stores the user's entered budget amount after validation.

Constructor

Initializes the screen with information about the budget preference.

public UI_Screen5(boolean isBudgetYesSelected)

Methods

start(Stage primaryStage)

The main entry point for the screen. Sets up the UI components and handles the following functionality:

  • Prompts the user to enter a budget amount.
  • Validates that the entered amount is a valid integer.
  • Displays feedback on whether the input is valid or invalid.
  • Provides a "Continue" button for the user to proceed or complete the process.
public void start(Stage primaryStage)

User Interaction

  • The user enters a budget amount in the text field.
  • If the input is valid, a green feedback message confirms the budget.
  • If the input is invalid, a red feedback message prompts the user to enter a valid integer.
  • Pressing the Continue button triggers the following actions:
    • If isBudgetYesSelected is true, the screen closes, indicating a transition to the next step (not implemented in this class).
    • If isBudgetYesSelected is false, a success message is displayed, and the application exits.
  • The "Enter" key is linked to the submit button for convenience.

Validation

The following validations are implemented:

  • The budget input must be a valid integer.
  • Feedback is displayed immediately after the user submits the input.

UI Layout

The user interface uses the following JavaFX components:

  • Label: Displays the question prompt and feedback messages.
  • TextField: Allows the user to input their budget.
  • Button: Used for submitting the budget and continuing to the next step.
  • VBox: Organizes the components vertically with spacing.
  • HBox: Aligns the Continue button at the bottom-right.

Navigation Logic

  • On Submit, validates the budget and updates the feedback message.
  • On Continue, the following occurs:
    • If traveling on a budget (isBudgetYesSelected), the screen transitions to the next step.
    • Otherwise, a success message is displayed, and the application exits.

Debugging Information

The screen prints the following debugging information to the console:

  • The entered budget amount upon successful validation.
