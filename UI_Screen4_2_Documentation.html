Class Documentation: UI_Screen4_2

The UI_Screen4_2 class is a JavaFX screen where users define starting and finish points for the selected change days in their journey.

Class Overview

This screen provides functionality for editing starting and finish points for each day the user has chosen to change their trailhead. It uses ObservableList to manage the dynamic data for the starting and finish points.

Fields

  • ObservableList<Integer> changeDays: The days when the trailhead changes, passed from the previous screen.
  • ObservableList<String> startingPoints: Stores starting points for each change day.
  • ObservableList<String> finishPoints: Stores finish points for each change day.
  • boolean isBudgetYesSelected: Indicates whether the user is traveling on a budget.

Constructor

Initializes the fields with data passed from the previous screen and ensures startingPoints and finishPoints have entries matching the size of changeDays.

public UI_Screen4_2(ObservableList<Integer> changeDays, boolean isBudgetYesSelected)

Methods

start(Stage primaryStage)

The main entry point for the screen. Sets up the UI components and handles the following functionality:

  • Displays a table of change days.
  • Allows users to input starting and finish points for each change day.
  • Ensures finish points default to the corresponding starting point if left empty.
  • Transitions to the next screen or completes the process based on the budget preference.
public void start(Stage primaryStage)

User Interaction

  • The user enters starting and finish points in text fields aligned with each change day.
  • Leaving a finish point empty automatically assigns the corresponding starting point.
  • Clicking Continue triggers the following actions:
    • Missing finish points are replaced with starting points.
    • If isBudgetYesSelected is true, the screen transitions to UI_Screen5.
    • If isBudgetYesSelected is false, a success message is displayed, and the application exits.

Validation

The following validations are implemented:

  • Finish points default to starting points if left empty.
  • All lists (changeDays, startingPoints, and finishPoints) maintain synchronized lengths.

UI Layout

The user interface uses the following JavaFX components:

  • TableView: Displays the change days.
  • VBox: Organizes text fields for starting and finish points.
  • Button: Triggers the transition to the next screen or completion of the process.
  • HBox: Aligns the table and text fields horizontally.

Navigation Logic

  • On Continue, data is finalized, and the following occurs:
    • If traveling on a budget (isBudgetYesSelected), proceeds to UI_Screen5.
    • Otherwise, displays a success message and closes the application.

Debugging Information

The screen prints the following debugging information to the console:

  • Change days list.
  • Starting points and finish points lists upon pressing Continue.
